# Проверить на коллизии

*Доступно с версии 1.0.2*

⚠Для работы с помещениями (категория Room) требуется Renga Professional (из-за необходимости считывать геометрию помещений из IFC). В противном случае они не будут проанализированы (функция отработает без ошибок)

Применять функцию можно для двух основных задач:

* проверка групп объектов выбранных категорий между собой на взаимные геометрические пересечения с допуском с автоматическим формированием отчета вида (сырые данные):
  
  ```xml
      <ClashDetectiveReportItem>
        <Date>27.09.2025 18:52:26</Date>
        <Relation>Intersecting</Relation>
        <NameObject1>Стена: 400.00 мм</NameObject1>
        <ObjectId1>d234eaa0-8f6a-28ab-f6f1-e8f78d9c9972</ObjectId1>
        <CategoryObject1>Wall</CategoryObject1>
        <NameObject2>Перекрытие: 150.00 мм</NameObject2>
        <ObjectId2>a638e202-d77a-dcbf-3305-3f487d09b052</ObjectId2>
        <CategoryObject2>Floor</CategoryObject2>
      </ClashDetectiveReportItem>
  ```

* возможность скопировать свойства (**только** свойства, не атрибуты, расчетные характеристики и параметры) от объекта первой группы к объекту второй группы, если второй объект как-либо геометрически связан с проверяемым объектом из первой группы.

Окно утилиты после запуска выглядит следующим образом:

![](assets/2025-09-28-11-29-00-image.png)

Необходимыми данными для проверки являются выбранные категории (одна или несколько) для первой и второй группы. Выделенная категория подсвечивается голубой или серой подсветкой (голубой, если выделение задано в данной сессии утилиты, а серым, если параметры загружены из конфига в памяти). Для снятия выделения необходимо повторно щёлкнуть ЛКМ на позиции списка.

Критерии проверки геометрии позволяют выделить один или несколько критериев проверки. Если приведенные к триангуляции 3D-тела будут иметь хотя бы одно из видов пересечений, они будут считаться коллизирующими (попадать в отчет и передавать друг другу свойства, если данная опция выбрана).

Перечень параметров для копирования настраивается в случае необходимости связки коллизирующих объектов друг с другом. Так как отношение связи может быть "один ко многим", то заполняться будет для одного из объектов. Связывание параметров оставлено только для объектов второй группы по отношению к первой (для обратного процесса измените выбранные категории между группами вручную, TODO - <u>кнопка смены выбранных категорий</u>)

Настроенный конфиг можно сохранить отдельно, как и загрузить его из файла. Папка по умолчанию, открывающаяся при открытии\сохранении - папка с конфигами приложения на ПК.

При нажатии на "Начать расчет", если в сравниваемых группах выбрана хотя бы одна категория, текущий конфиг будет принудительно сохранен под именем Default.xml в папку приложения и загружен впоследствии при новом вызове данной утилиты.

Для просмотра отчета по найденным коллизиям используйте [команду "Посмотреть отчет о коллилиях"](.\6.2_RENGA_BRI4KA_COLLISIONSVIEWER.md).
